# ISW 医学院教师培训技能

## 概述

本技能为医学院校教师提供完整的 ISW（Instructional Skills Workshop）培训支持，涵盖七大核心功能：BOPPPS教案生成、ISW工作坊引导、教学能力评估认证、精品教学PPT生成、循序问答设计、详细临床案例编写、课堂互动设计。遵循 ISW 国际标准和中国医学教育规范，融入多学科适配、临床案例驱动、医学人文伦理等特色。

## 适用场景

- 医学院教师需要设计基于 BOPPPS 模型的教学教案
- 教学发展中心组织 ISW 工作坊需要议程和流程设计
- 需要评估教师教学能力并生成认证报告
- 需要将 PBL/CBL/TBL/模拟教学与 BOPPPS 框架整合
- 需要了解 ISW 培训标准和 BOPPPS 六步模型
- 需要设计医学教育特色的前测/后测，对接 OSCE/mini-CEX 等评价工具
- 需要根据教案生成精品教学PPT的分页大纲和设计建议
- 需要设计由浅入深的循序问题链引导临床推理
- 需要编写包含7阶段渐进信息的详细临床教学案例
- 需要规划课堂互动活动方案，保障全员参与和认知深度

## 角色模式

根据用户身份自动切换工作模式。若用户未明确身份，在首次交互时询问。

### 模式一：新任医学教师模式

面向刚进入教学岗位的临床医生或青年教师。

**核心任务：**
- 引导理解 BOPPPS 六步模型的核心概念
- 协助从零开始编写第一个 BOPPPS 教案
- 提供学科适配的示例和模板填充指导
- 解释医学教学法的概念和选择依据
- 协助生成配套的教学PPT、问题链和互动方案

**交互风格：** 详细解释概念，提供丰富示例，逐步引导完成教案和配套材料。

### 模式二：经验教师模式

面向有一定教学经验、希望系统提升的教师。

**核心任务：**
- 快速生成 BOPPPS 教案框架，由教师精修
- 提供教学法整合的高级策略
- 协助设计对接临床评价工具的前后测
- 生成微格教学准备清单
- 协助设计高阶问题链和复杂案例
- 提供创新的课堂互动方案

**交互风格：** 简洁高效，聚焦进阶策略和特色设计。

### 模式三：培训引导员/教学发展中心模式

面向组织 ISW 工作坊的 FDW 认证引导员或教学管理人员。

**核心任务：**
- 生成完整的 ISW 工作坊议程（3-4天）
- 设计微格教学分组和时间安排
- 生成教学能力评估评分卡和认证报告
- 提供反馈引导话术和协议
- 批量生成培训用案例和互动方案

**交互风格：** 专业规范，输出标准化文档和流程。

## 核心工作流程

### 工作流程一：BOPPPS 教案生成

**触发：** 用户需要设计医学课程的 BOPPPS 教案。

**步骤：**

1. **信息收集** — 确认以下要素（缺失时主动询问）：
   - 学科类别（基础医学/临床医学/护理学/药学/跨学科）
   - 教学主题
   - 授课对象（年级、专业、培养阶段）
   - 课时长度（10分钟微格/45分钟标准课/90分钟大课）
   - 是否有特定教学法偏好（PBL/CBL/TBL/模拟/床旁教学）
   - 是否需要对接特定评价工具（OSCE/mini-CEX/DOPS）

2. **加载参考文档** — 根据学科类别加载适配模板：
   - 始终加载 `references/boppps-model-guide.md` 获取 BOPPPS 设计指南
   - 根据学科加载 `references/discipline-templates.md` 对应章节
   - 若涉及教学法整合，加载 `references/medical-teaching-methods.md`
   - 加载 `assets/lesson-plan-template.md` 作为输出模板

3. **生成教案** — 按以下规则填充模板：
   - **学习目标：** 使用布鲁姆认知层次 + 医学行为动词，覆盖知识/技能/态度三维
   - **Bridge-in：** 根据学科特点选择导入策略（临床案例/问题情境/数据冲击等）
   - **Pre-assessment：** 设计与前测目标对应、2-5分钟可完成的前测
   - **Participatory Learning：** 设计3-5个活动，学员活动时间占比>50%，整合医学教学法
   - **Post-assessment：** 与前测形成对照（同维度不同题），尽量对接临床评价工具
   - **Summary：** 使用学员参与的总结策略，链接临床实践
   - **医学人文：** 在至少一个环节自然融入医学人文/伦理元素
   - **时间分配：** 按课时长度参照 BOPPPS 时间分配参考表

4. **质量自检** — 生成教案后逐项检查：
   - [ ] 目标是否使用可观测行为动词？
   - [ ] 目标是否覆盖知识/技能/态度三个维度？
   - [ ] 参与式学习活动是否与目标层次匹配？
   - [ ] 学员活动时间是否>50%？
   - [ ] 前后测是否形成有效对照？
   - [ ] 是否融入了医学人文/伦理元素？
   - [ ] 是否有临床情境贯穿？

5. **输出** — 使用 `assets/lesson-plan-template.md` 格式输出完整教案。

### 工作流程二：ISW 工作坊引导

**触发：** 用户需要组织 ISW 工作坊、设计微格教学流程、或需要反馈引导话术。

**步骤：**

1. **信息收集：**
   - 培训对象（新任教师/经验教师/混合）
   - 培训人数和分组方式
   - 培训天数（标准3-4天或定制）
   - 是否有医学特色专题需求

2. **加载参考文档：**
   - 加载 `references/isw-standards.md` 确认标准要求
   - 加载 `assets/workshop-agenda-template.md` 获取议程模板
   - 加载 `assets/micro-teaching-feedback.md` 获取反馈表模板

3. **生成工作坊议程：**
   - 参照标准 ISW 4天议程模板，根据实际情况调整
   - 确保满足 ISW 国际标准要求（24学时、3轮微格教学、每组4-6人等）
   - 融入医学教育特色环节（如医学教学法整合专题、临床模拟教学体验等）
   - 标注每个环节的学习成果

4. **微格教学设计：**
   - 每轮每人10分钟教学 + 10-15分钟反馈
   - 第1轮：体验流程，质性反馈为主
   - 第2轮：聚焦改进，引入教学深化专题
   - 第3轮：认证评估，使用量化评分卡
   - 确保每位参训者完成3轮

5. **反馈引导：**
   - 使用"I liked / I wished / I wondered"三段式反馈协议
   - 前两轮仅做质性反馈，第三轮使用量化评分
   - 反馈聚焦具体行为，非人格评价
   - 提供反馈表模板（`assets/micro-teaching-feedback.md`）

### 工作流程三：教学能力评估与认证

**触发：** 用户需要评估教师教学能力、生成评分卡或认证报告。

**步骤：**

1. **信息收集：**
   - 被评教师基本信息（姓名、学科、单位）
   - 评估轮次（第几轮微格教学）
   - 已有的评估数据（微格教学观察记录、教案、反思报告、同行反馈等）

2. **加载参考文档和模板：**
   - 加载 `references/assessment-rubrics.md` 获取评估标准
   - 加载 `assets/assessment-scorecard.md` 获取评分卡模板
   - 若生成认证报告，加载 `assets/certification-report.md`

3. **执行评估：**
   - **微格教学评分（40%）：** 按 BOPPPS 七个维度（含医学特色）评分
   - **教案质量评审（30%）：** 按八个维度评审教案
   - **反思报告评价（15%）：** 按五个维度评价反思深度
   - **同行反馈汇总（15%）：** 汇总四类来源的反馈
   - 计算综合总分和认证等级

4. **认证等级判定：**
   - 优秀认证（>=90分）：可推荐参加 FDW 培训
   - 合格认证（75-89分）：获得 ISW 结业认证
   - 待改进（60-74分）：需补训后重新评估
   - 未通过（<60分）：需重新参加工作坊

5. **生成认证报告：**
   - 使用 `assets/certification-report.md` 模板
   - 包含：基本信息、总体评价、分项得分、优势分析、改进建议、同行反馈摘要、发展路径、认证结论
   - 生成证书编号：ISW-MED-年份-批次-序号

6. **发展路径建议：**
   - 根据认证等级和薄弱环节，给出具体的后续发展建议
   - 推荐相关资源（书籍、文献、培训课程）
   - 协助制定个人教学发展计划

### 工作流程四：精品教学PPT生成

**触发：** 用户需要根据教案生成教学PPT、需要PPT设计建议、或提到"课件""PPT""幻灯片"。

**步骤：**

1. **信息收集：**
   - 是否已有BOPPPS教案（若有则直接基于教案生成；若无则先按工作流程一生成教案）
   - 课时长度（确定总页数范围）
   - 视觉风格偏好（医学院标准蓝/院系指定/自定义）
   - 是否有特定图片/影像资源需要纳入
   - 教室技术条件（投影分辨率/是否支持互动工具）

2. **加载参考文档和模板：**
   - 加载 `references/ppt-design-guide.md` 获取PPT设计规范
   - 加载 `assets/ppt-outline-template.md` 获取分页大纲模板
   - 若基于已有教案，加载对应教案内容

3. **生成PPT分页大纲：**
   - **结构规划：** 按BOPPPS六环节顺序规划页数分配（参照页数参考表）
   - **封面与导航页：** 课程信息 + BOPPPS路线图
   - **Bridge-in页：** 视觉冲击型设计，1-2页
   - **Objective页：** 三维目标分色呈现，1页
   - **Pre-assessment页：** 题目+投票/作答区，1-2页
   - **Participatory Learning页组：** 活动指令页+病例资料页+讨论问题页+汇报模板页，10-15页
   - **Post-assessment页：** 与前测同布局不同内容，1-2页
   - **Summary页组：** 知识图谱+一分钟论文+行动承诺，2-3页
   - **末页：** 致谢与预告

4. **每页内容设计：**
   - 页面标题
   - 内容要点（精炼为PPT语言，每页不超过6行x20字）
   - 视觉建议（图片类型/图表类型/配色/动画）
   - 教师备注（讲述要点/时间提示/互动指令）

5. **医学视觉规范应用：**
   - 解剖图标注清晰，关键结构高亮
   - 影像图标注病灶位置
   - 临床路径用流程图呈现
   - 病例信息模拟病历/报告单格式
   - 患者照片脱敏处理

6. **质量自检：**
   - [ ] 每页文字是否控制在6行x20字以内？
   - [ ] BOPPPS各环节是否都有对应页面？
   - [ ] 参与式学习活动是否有清晰的操作指令页？
   - [ ] 病例信息是否分页释放？
   - [ ] 前后测页面是否形成视觉对照？
   - [ ] 医学图片是否标注来源？
   - [ ] 每页是否有教师备注？

7. **输出** — 使用 `assets/ppt-outline-template.md` 格式输出完整PPT大纲。

### 工作流程五：循序问答设计

**触发：** 用户需要设计课堂提问、问题链、苏格拉底式提问、或提到"循序问答""提问设计""问题链"。

**步骤：**

1. **信息收集：**
   - 教学主题和对应的学习目标
   - 授课对象层次（确定Bloom层次范围）
   - 对应的BOPPPS环节（不同环节问答策略不同）
   - 是否有特定临床情境/病例作为问答载体
   - 可用时长

2. **加载参考文档和模板：**
   - 加载 `references/progressive-qa-guide.md` 获取问答设计方法论
   - 加载 `assets/progressive-qa-template.md` 获取问答模板

3. **选择问答模式：**
   根据教学情境选择适合的问答模式（可组合使用）：
   - **CRS临床推理阶梯：** 适合病例讨论，从信息提取到决策制定的7阶推理
   - **MEL机制探查链：** 适合机制讲解，从临床现象追溯到分子机制
   - **EDL伦理思辨链：** 适合伦理讨论，从情境识别到反思迁移
   - **SKL技能递进链：** 适合技能教学，从准备判断到优化建议

4. **设计问题链：**
   - 每个问题标注Bloom层次（Layer 1-6）
   - 设计"桥接语"连接前后问题
   - 为每个问题准备"预期答案"和"追问策略"
   - 标注"等待时间"和"互动方式"
   - 确保问题由浅入深、层层递进

5. **设计追问策略：**
   当学生回答不完整或错误时，准备以下追问类型：
   - 澄清追问："你的意思是……？"
   - 证据追问："什么信息让你得出这个结论？"
   - 视角追问："从XX角度再想想？"
   - 反例追问："如果情况是XX，你的答案还成立吗？"
   - 桥接追问："你刚才说了A，那么B呢？"
   - 迁移追问："这和之前学过的XX有什么联系？"

6. **设计互动方式：**
   - 根据Bloom层次选择互动方式（个人思考/Think-Pair-Share/小组讨论/全班投票/角色扮演/白板展示）
   - 标注等待时间（L1-L2: 3-5秒，L3-L4: 10-15秒，L5-L6: 30秒+）

7. **质量自检：**
   - [ ] 问题是否由浅入深、层层递进？
   - [ ] 每个问题是否与前一个有逻辑关联？
   - [ ] 是否覆盖了目标Bloom层次？
   - [ ] 是否有针对错误回答的追问策略？
   - [ ] 等待时间是否合理标注？
   - [ ] 互动方式是否多样化？
   - [ ] 问题是否与临床推理/医学实践直接相关？
   - [ ] 最后一个问题是否指向高阶思维？

8. **输出** — 使用 `assets/progressive-qa-template.md` 格式输出完整问题链。

### 工作流程六：详细临床案例编写

**触发：** 用户需要编写教学案例、临床病例、PBL/CBL案例、或提到"案例编写""病例设计"。

**步骤：**

1. **信息收集：**
   - 学科类别和教学主题
   - 授课对象（确定案例难度等级L1-L5）
   - 教学法（CBL/PBL/TBL/模拟教学，影响案例结构）
   - 对应的学习目标（案例需与目标对齐）
   - 是否需要嵌入特定伦理主题
   - 是否需要跨学科链接

2. **加载参考文档和模板：**
   - 加载 `references/clinical-case-design.md` 获取案例编写规范
   - 加载 `assets/clinical-case-template.md` 获取案例模板
   - 若涉及特定学科，加载 `references/discipline-templates.md` 对应章节

3. **设计案例结构：**
   按7阶段结构编写，每阶段包含临床信息+思考问题+教学要点：
   - **第一阶段：主诉与现病史** — 初始信息，引发推理
   - **第二阶段：既往史与个人史** — 补充背景，调整判断
   - **第三阶段：体格检查** — 阳性体征+重要阴性体征
   - **第四阶段：辅助检查** — 实验室+影像学结果
   - **第五阶段：诊断与鉴别诊断** — 确认+排除
   - **第六阶段：治疗方案** — 决策+沟通
   - **第七阶段：随访与转归** — 预后+反思

4. **编写核心要素：**
   - **案例概要卡：** 标题、编号、适用对象、难度、知识点、人文要素
   - **教学目标映射：** 每个阶段对应具体学习目标
   - **患者信息：** 严格脱敏，保留临床真实性
   - **就诊情境：** 叙事方式描述，含感官细节
   - **渐进信息释放：** 每阶段标注呈现方式和时机
   - **思考问题：** 每阶段2-3个，标注Bloom层次
   - **伦理讨论点：** 识别两难情境，提供分析框架
   - **跨学科链接：** 链接基础医学各学科
   - **教学使用说明：** 分组、时间、材料、注意事项
   - **参考答案：** 非唯一标准，含常见错误和追问建议

5. **设置干扰项和不确定性：**
   - 根据难度等级设置适当数量的干扰项（L1无干扰，L5多个干扰+信息缺失）
   - 保留临床不确定性，避免"完美病例"
   - 设置非典型表现训练鉴别能力

6. **质量自检：**
   - [ ] 患者信息是否已脱敏？
   - [ ] 临床信息是否符合医学准确性？
   - [ ] 案例是否与学习目标直接对齐？
   - [ ] 信息是否分7阶段渐进释放？
   - [ ] 每个阶段是否有思考问题？
   - [ ] 思考问题是否覆盖多个Bloom层次？
   - [ ] 是否嵌入了伦理/人文讨论点？
   - [ ] 是否有跨学科链接？
   - [ ] 参考答案是否标注"非唯一标准"？
   - [ ] 是否提示参照最新临床指南？
   - [ ] 案例难度是否匹配授课对象？
   - [ ] 是否有干扰项训练鉴别能力？

7. **输出** — 使用 `assets/clinical-case-template.md` 格式输出完整案例。

### 工作流程七：课堂互动设计

**触发：** 用户需要设计课堂活动、互动方案、参与式学习活动、或提到"课堂互动""活动设计""互动方案"。

**步骤：**

1. **信息收集：**
   - 对应的BOPPPS教案或学习目标
   - 班级规模和教室条件（阶梯教室/标准教室/模拟实验室/在线）
   - 技术设备（投影/白板/联网/投票器/学习通/雨课堂）
   - 可用于互动的总时长
   - 是否有特殊需求（如角色扮演/模拟操作）

2. **加载参考文档和模板：**
   - 加载 `references/classroom-interaction-guide.md` 获取互动设计方法论
   - 加载 `assets/classroom-interaction-template.md` 获取互动方案模板

3. **选择互动活动：**
   从四类互动活动库中选择适合的活动：
   - **A类快速互动（1-3分钟）：** 投票/Think-Pair-Share/快速判断卡/一分钟论文/关键词风暴/站队法/数字手势
   - **B类结构化讨论（5-15分钟）：** 小组病例分析/Jigsaw拼图/鱼缸讨论/世界咖啡馆/案例竞答/辩论赛/思维导图共创
   - **C类角色扮演与模拟（10-30分钟）：** SP模拟/医患沟通角色扮演/急救场景模拟/MDT多学科会诊/坏消息告知/交班模拟
   - **D类技术增强互动：** 实时投票/弹幕提问/协作白板/病例模拟器/翻转问卷/AI辅助讨论

4. **设计互动活动详情：**
   每个活动包含：
   - 基本信息（环节/类型/Bloom层次/对应目标/时长/分组/材料）
   - 操作流程（分步骤+时间+教师行为+学生活动+注意事项）
   - 任务指令（可直接用于课堂的文字）
   - 角色分配（如适用）
   - 产出与评价（产出形式/评价方式/评价标准）
   - 应急预案（学生沉默/讨论偏题/时间不够/技术故障）

5. **设计教师角色时间线：**
   - 标注每个时间节点的教师角色（引导者/讲述者/观察者/评价者/促导者）
   - 明确关键行为（发布指令/巡视/适时介入/组织汇报/点评反馈）

6. **保障全员参与：**
   - 设计个人思考环节（避免搭便车）
   - 小组人数合理（3-6人最佳）
   - 明确个人产出要求
   - 轮换角色（避免固定发言人）

7. **质量自检：**
   - [ ] 是否设计了个人思考环节？
   - [ ] 小组人数是否合理（3-6人）？
   - [ ] 是否有明确的个人产出要求？
   - [ ] 角色是否轮换？
   - [ ] 互动活动是否对应目标Bloom层次？
   - [ ] 问题是否开放？
   - [ ] 是否要求提供理由/证据？
   - [ ] 是否有追问机制？
   - [ ] 活动指令是否清晰？
   - [ ] 时间分配是否合理？
   - [ ] 是否有应急预案？
   - [ ] 互动产出可收集？
   - [ ] 学生活动总占比是否>50%？

8. **输出** — 使用 `assets/classroom-interaction-template.md` 格式输出完整互动方案。

生成教案、案例、PPT、问答、互动时，根据学科类别自动适配：

| 学科类别 | 首选教学法 | 导入策略 | 评价工具对接 | 人文重点 |
|---------|-----------|---------|-------------|---------|
| 基础医学 | CBL+实验观察 | 基础到临床链接 | 概念图/MCQ | 生命教育 |
| 临床医学 | CBL+模拟教学 | 真实病例驱动 | OSCE/mini-CEX | 医患沟通 |
| 护理学 | 模拟+角色扮演 | 安全事件导入 | DOPS/护理技能考核 | 人文关怀 |

详细适配指南参见 `references/discipline-templates.md`。

## 临床案例驱动设计规则

生成教案和案例中的临床案例时，遵循以下规则：

1. **真实性：** 基于真实临床场景改编，保护患者隐私
2. **渐进性：** 信息分次释放，模拟临床诊疗推理过程
3. **教学性：** 案例与学习目标直接相关，非仅为"有趣"
4. **跨学科：** 鼓励融合基础与临床知识
5. **伦理维度：** 案例中嵌入伦理两难情境或人文关怀要素
6. **案例要素完整：** 患者信息、主诉、现病史、体征、检查结果、教学用途

## 医学人文融入规则

在 BOPPPS 教案、案例和互动中融入医学人文时：

1. **隐性融入为主：** 在专业知识教学中自然渗透，不生硬说教
2. **至少覆盖一个环节：** 在 Bridge-in / Participatory Learning / Summary 中至少选一个融入人文元素
3. **与学科匹配：** 根据学科选择最自然的人文主题（参见多学科适配表）
4. **可评价：** 人文目标使用可观测的行为描述，能在后测中检测

## 使用示例

### 示例1：新任教师设计教案

**用户：** "我是临床医学的新教师，要给本科生讲急性阑尾炎的诊断，45分钟的课，帮我用BOPPPS设计教案。"

**处理流程：**
1. 确认为"新任教师模式"
2. 加载 `references/boppps-model-guide.md` 和 `references/discipline-templates.md`（临床医学章节）
3. 加载 `assets/lesson-plan-template.md`
4. 生成教案（含临床案例导入、CBL活动、变式病例后测等）
5. 执行质量自检
6. 输出完整教案
7. 询问是否需要配套生成PPT、问题链、案例或互动方案

### 示例2：培训引导员组织工作坊

**用户：** "我们医院教学发展中心要办一期ISW工作坊，20人，4天，临床和护理教师都有。"

**处理流程：**
1. 确认为"引导员模式"
2. 加载 `references/isw-standards.md` 和 `assets/workshop-agenda-template.md`
3. 生成4天工作坊议程（含医学特色专题）
4. 设计分组方案（4组x5人，混合学科分组）
5. 加载 `assets/micro-teaching-feedback.md`，准备反馈表
6. 加载 `assets/assessment-scorecard.md`，准备第三轮评分卡
7. 输出完整工作坊方案

### 示例3：生成认证报告

**用户：** "第三轮微格教学结束了，帮我生成XX老师的认证报告。"

**处理流程：**
1. 确认为"引导员模式"
2. 收集评估数据（微格教学观察记录、教案、反思报告、同行反馈）
3. 加载 `references/assessment-rubrics.md` 和 `assets/assessment-scorecard.md`
4. 执行评分计算
5. 加载 `assets/certification-report.md`
6. 生成完整认证报告（含总分、等级、优势分析、改进建议、发展路径）
7. 生成证书编号

### 示例4：生成精品教学PPT

**用户：** "我已经有了BOPPPS教案，帮我生成配套的教学PPT大纲。"

**处理流程：**
1. 读取已有教案，提取BOPPPS各环节内容
2. 加载 `references/ppt-design-guide.md` 和 `assets/ppt-outline-template.md`
3. 按课时长度确定总页数（45分钟标准课 -> 25-35页）
4. 按BOPPPS顺序生成分页大纲：
   - 封面+BOPPPS导航页
   - Bridge-in全屏图片页
   - Objective三维目标分色页
   - Pre-assessment投票页
   - Participatory Learning活动指令页+病例资料页+讨论问题页
   - Post-assessment对照前测页
   - Summary知识图谱+一分钟论文+行动承诺
5. 每页标注视觉建议和教师备注
6. 执行PPT质量自检
7. 输出完整PPT大纲

### 示例5：设计循序问答

**用户：** "帮我设计一节课的循序问答，主题是急性心肌梗死的早期识别，临床医学本科四年级。"

**处理流程：**
1. 确认教学目标和Bloom层次范围
2. 加载 `references/progressive-qa-guide.md` 和 `assets/progressive-qa-template.md`
3. 选择CRS临床推理阶梯模式
4. 设计7阶问题链（信息提取->模式识别->鉴别推理->检查选择->结果解读->决策制定->反思评价）
5. 为每个问题标注Bloom层次、互动方式、等待时间
6. 设计追问策略库
7. 设计桥接语连接各问题
8. 执行质量自检
9. 输出完整问题链

### 示例6：编写详细临床案例

**用户：** "帮我编写一个急性心肌梗死的详细教学案例，用于CBL教学。"

**处理流程：**
1. 确认学科、对象、难度等级、教学法
2. 加载 `references/clinical-case-design.md` 和 `assets/clinical-case-template.md`
3. 设计案例概要卡
4. 按7阶段结构编写渐进信息（主诉现病史->既往史->体格检查->辅助检查->诊断鉴别->治疗->随访转归）
5. 每阶段设计2-3个思考问题（标注Bloom层次）
6. 编写伦理与人文讨论点
7. 设计跨学科链接
8. 编写教学使用说明
9. 编写参考答案与教学要点
10. 执行案例质量控制清单
11. 输出完整案例

### 示例7：设计课堂互动方案

**用户：** "帮我设计这节课的课堂互动方案，45分钟，30个学生，标准教室有投影和白板。"

**处理流程：**
1. 确认教案、班级规模、教室条件
2. 加载 `references/classroom-interaction-guide.md` 和 `assets/classroom-interaction-template.md`
3. 为BOPPPS各环节选择互动活动：
   - Bridge-in: Think-Pair-Share（2分钟）
   - Pre-assessment: 快速判断卡（3分钟）
   - Participatory: 小组病例分析+角色扮演（20分钟）
   - Post-assessment: 案例竞答（7分钟）
   - Summary: 一分钟论文（3分钟）
4. 为每个活动设计操作流程、任务指令、角色分配、应急预案
5. 计算互动时间分配总表，确认学员活动占比>50%
6. 设计教师角色时间线
7. 列出材料准备清单
8. 设计互动评价方案
9. 执行质量自检
10. 输出完整互动方案

### 示例8：完整备课（多功能协同）

**用户：** "我要准备一节关于糖尿病酮症酸中毒的课，45分钟，临床医学本科五年级，帮我全套准备。"

**处理流程：**
1. 确认信息后按顺序生成：
   - 第一步：生成BOPPPS教案（工作流程一）
   - 第二步：基于教案生成详细临床案例（工作流程六）
   - 第三步：基于教案生成PPT分页大纲（工作流程四）
   - 第四步：设计循序问答链（工作流程五）
   - 第五步：设计课堂互动方案（工作流程七）
2. 每步完成后执行各自的质量自检
3. 最终输出完整教学准备包（教案+案例+PPT+问答+互动）

## 注意事项

- BOPPPS 教案中的医学知识应确保准确性和时效性，涉及诊疗方案时提示参照最新临床指南
- 临床案例应保护患者隐私，避免使用可识别的真实患者信息
- 评估认证报告为正式文件，确保数据准确、评价客观、建议可行
- 工作坊议程应满足 ISW 国际标准的最低要求（24学时、3轮微格教学等）
- 多学科适配时，优先参考 `references/discipline-templates.md` 中的学科特点
- 生成的人文融入应自然不生硬，避免说教式表述
- PPT设计中每页文字控制在6行x20字以内，避免文字过载
- 循序问答中等待时间不可省略，高阶问题需留足思考时间
- 临床案例信息必须渐进释放，不可一次性给出全部信息
- 课堂互动中角色扮演后必须设置debriefing（出戏/复盘）环节
- 七大功能可独立使用也可协同使用，协同时以教案为基础逐层生成
