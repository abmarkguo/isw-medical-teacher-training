# BOPPPS 六步教学模型详解

## 一、BOPPPS 模型概述

BOPPPS 是 ISW 工作坊的核心教学设计框架，将一节课（通常10-50分钟）分为六个连贯环节。每个环节有明确的目标和设计原则。

**六步全称：**
- **B**ridge-in（导入）
- **O**bjective / **O**utcome（目标/学习成果）
- **P**re-assessment（前测）
- **P**articipatory Learning（参与式学习）
- **P**ost-assessment（后测）
- **S**ummary（总结）

## 二、各环节详解

### 2.1 Bridge-in（导入）

**目标：** 吸引注意力，激活先备知识，建立学习动机

**设计原则：**
- 时长控制在本节课的5-10%
- 与学习目标直接相关，不做"标题党"
- 激发认知冲突或情感共鸣

**医学教育常用导入策略：**

| 策略 | 说明 | 医学示例 |
|------|------|---------|
| **临床案例导入** | 用真实/改编病例引发思考 | "35岁女性，突发胸痛2小时来诊……" |
| **问题情境导入** | 提出两难问题激发探究 | "如果患者拒绝输血，作为医生你怎么办？" |
| **数据冲击导入** | 用震撼数据引发关注 | "全球每年因用药错误死亡的人数超过交通事故" |
| **实物/标本导入** | 展示病理标本或影像 | 展示一张典型心电图让学员诊断 |
| **角色扮演导入** | 模拟医患场景 | 学员扮演标准化病人和医生 |
| **新闻事件导入** | 用医学热点新闻切入 | 近期公共卫生事件分析 |

### 2.2 Objective / Outcome（目标/学习成果）

**目标：** 明确告知学员本节课"能做什么"

**设计原则：**
- 使用行为动词（可观测、可评价）
- 包含四个维度：ABCD（Audience对象、Behavior行为、Condition条件、Degree标准）
- 目标数量3-5个为宜
- 区分知识、技能、态度三个层次

**医学教育目标编写框架：**

**布鲁姆认知层次 + 医学行为动词：**

| 认知层次 | 行为动词 | 医学示例 |
|---------|---------|---------|
| 记忆 | 列出、说出、识别 | 列出急性心肌梗死的五大典型症状 |
| 理解 | 解释、描述、归纳 | 解释心电图ST段抬高的病理生理机制 |
| 应用 | 实施、执行、操作 | 按照心肺复苏指南实施成人基础生命支持 |
| 分析 | 鉴别、比较、分析 | 鉴别急性胸痛的三大常见病因 |
| 评价 | 评估、判断、论证 | 评估患者是否符合静脉溶栓适应症 |
| 创造 | 设计、制定、构建 | 为给定病例制定个体化治疗方案 |

**医学三维度目标示例：**
- **知识目标：** 能准确说出急性阑尾炎的典型临床表现和鉴别诊断要点
- **技能目标：** 能在模拟人上正确完成麦氏点压痛和反跳痛的体格检查
- **态度目标：** 能体认到急腹症患者焦虑情绪并给予适当的人文关怀

### 2.3 Pre-assessment（前测）

**目标：** 了解学员已有知识水平，调整教学节奏

**设计原则：**
- 快速（2-5分钟）
- 与学习目标直接对应
- 结果即时反馈给学员和教师

**医学教育前测方法：**

| 方法 | 适用场景 | 操作方式 |
|------|---------|---------|
| **选择题快速投票** | 知识类目标 | 2-3道MCQ，举手/扫码答题 |
| **概念图填空** | 理解类目标 | 给出框架，学员补充关键词 |
| **病例判断** | 临床推理类目标 | 给出简短病例，让学员判断诊断方向 |
| **自我评估量表** | 技能类目标 | "你对XXX操作的自信程度：1-5分" |
| **关键词头脑风暴** | 激活先备知识 | "提到糖尿病，你想到哪些关键词？" |

### 2.4 Participatory Learning（参与式学习）

**目标：** 学员主动参与学习过程，实现深度学习

**设计原则：**
- 本环节占整节课50-60%的时间
- 学员活动时间 > 教师讲授时间
- 活动设计与目标层次匹配
- 包含个人活动 + 小组活动

**医学教育参与式学习方法矩阵：**

| 目标层次 | 个人活动 | 小组活动 | 全班活动 |
|---------|---------|---------|---------|
| **知识层** | 阅读文献摘要、观看教学视频 | 知识卡片配对、概念图构建 | 快速问答竞赛 |
| **理解层** | 病例摘要分析、绘制流程图 | 病例讨论、Jigsaw拼图学习 | 案例辩论 |
| **应用层** | 模拟操作练习、开具医嘱 | 角色扮演（医患沟通）、TBL团队测验 | 临床技能演示 |
| **分析层** | 鉴别诊断推理、检验结果解读 | PBL小组讨论、多学科会诊模拟 | 复杂病例大讨论 |
| **评价层** | 方案评估报告、文献批判阅读 | 同行评议、临床决策讨论 | 伦理困境辩论 |
| **创造层** | 设计治疗方案、制作健康教育材料 | 设计临床路径、教学微课设计 | 质量改进方案展示 |

**与医学教学法的整合：**

- **PBL（Problem-Based Learning）：** 以临床问题为起点，学员自主探究。BOPPPS的参与式学习环节可完整嵌入一个PBL案例讨论周期。
- **CBL（Case-Based Learning）：** 以典型案例为载体，教师引导分析。适合BOPPPS中分析/评价层次的目标。
- **TBL（Team-Based Learning）：** 团队 readiness test → 应用活动。BOPPPS前测可对接TBL的iRAT/tRAT，参与式学习对接应用活动。
- **Simulation（模拟教学）：** 情景模拟 + Debriefing。BOPPPS的参与式学习和后测可整合模拟演练和复盘。

### 2.5 Post-assessment（后测）

**目标：** 检验学习成果是否达成目标

**设计原则：**
- 与前测形成对照（同维度不同题）
- 与学习目标一一对应
- 提供即时反馈

**医学教育后测方法：**

| 评估类型 | 方法 | 对接工具 |
|---------|------|---------|
| **知识掌握** | MCQ/简答题 | 可对接题库系统 |
| **临床推理** | 病例分析题 | 可对接 OSCE 笔试站 |
| **操作技能** | 操作观察评分 | 可对接 DOPS |
| **沟通能力** | 角色扮演评分 | 可对接 mini-CEX |
| **综合能力** | 模拟场景考核 | 可对接 OSCE 操作站 |
| **态度反思** | 反思日志/问卷 | 形成性评价 |

### 2.6 Summary（总结）

**目标：** 梳理要点，建立联系，延伸学习

**设计原则：**
- 时长5-10%
- 学员参与总结（非教师独讲）
- 连接下一节课或临床实践
- 留下"带走"的关键信息

**医学教育总结策略：**
- **一分钟论文：** "今天最重要的三点 + 仍有的一个疑问"
- **电梯演讲：** 用30秒向"同事"解释今天学到的核心概念
- **临床应用链接：** "明天在病房遇到这类患者，你会有什么不同？"
- **思维导图回顾：** 师生共同构建本节知识网络图
- **承诺行动：** 写下本周将在临床实践中尝试的一件事

## 三、BOPPPS 时间分配参考

| 环节 | 10分钟微格教学 | 45分钟标准课 | 90分钟大课 |
|------|:---:|:---:|:---:|
| Bridge-in | 1分钟 | 3-5分钟 | 5-8分钟 |
| Objective | 0.5分钟 | 2分钟 | 3分钟 |
| Pre-assessment | 1分钟 | 3-5分钟 | 5分钟 |
| Participatory Learning | 5分钟 | 20-25分钟 | 50-60分钟 |
| Post-assessment | 1.5分钟 | 5-8分钟 | 8-10分钟 |
| Summary | 1分钟 | 3-5分钟 | 5分钟 |

## 四、常见问题与避坑指南

| 问题 | 表现 | 改进建议 |
|------|------|---------|
| 导入与目标脱节 | 讲了精彩故事但未连接学习目标 | 设计导入时同步检查目标关联性 |
| 目标过大过多 | 一节课设定8-10个目标 | 精简至3-5个可操作目标 |
| 前测流于形式 | 只问"大家知道XXX吗？" | 改为具体可检测的题目 |
| 参与式学习=分组讨论 | 所有活动都是"讨论一下" | 根据目标层次匹配多样化活动 |
| 后测=前测重复 | 前后测题目完全相同 | 同维度、同难度、不同题 |
| 总结变重复 | 把讲过的内容再说一遍 | 转为学员主导的知识整合活动 |
